# 色付きカードの背景・文字色整合（Bootstrap 例準拠）

## DESIGN.md の修正は必要か

**必要（推奨）**。理由は次のとおりです。

- 実装は [assets/styles.css](c:\Users\ryone\Desktop\oshi-app\assets\styles.css) の `:root` トークン（`--app-card-*-text`）がデザインの正になりつつあるが、[DESIGN.md](c:\Users\ryone\Desktop\oshi-app\DESIGN.md) の「2.3 色付きカードのセット運用ルール」は **「必ずセット」** とある一方、**Secondary が「白文字面」なのか「薄面色＋本文色」なのか** が Bootstrap 例と一致して書かれていない。
- 今回の不具合（薄い `bg-secondary` に白文字）は、**仕様の曖昧さが実装の誤解釈を招いた**典型なので、ドキュメントで **Bootstrap 例との対応表** を1ブロック足すのが最も安全。

ドキュメント変更は「トークン表の1行修正＋対応表＋QA 1項目」程度に留め、バージョン番号（例: 1.2→1.3）を上げる想定。

---

## Step 1: 目標のペアリングを固定（Bootstrap 例と同じ意味）

ユーザー提示の HTML を **意味** で写し取る（クラス名はそのまま Dash では使わず、`card-main-*` に対応）。

| 意図 | Bootstrap 例 | アプリ側クラス | 文字色の方針 |
|------|----------------|----------------|--------------|
| 強い面色＋白文字 | `text-white bg-primary` 等 | `card-main-primary` / `info` / `success` / `danger` | 白系（`--bs-light` またはテーマの on-primary 相当） |
| 薄い面色＋濃い文字 | `bg-secondary`（`text-white` なし） | `card-main-secondary` | **本文色**（`--bs-body-color` または `--bs-emphasis-color`） |
| 注意（黄系） | 提示例は `text-white bg-warning` | `card-main-warning` | 現状は `text-dark` 相当。**提示例に合わせるなら白文字へ変更**（ただしテーマによってはコントラストが厳しいので QA で必ず確認） |
| 中立の薄い面 | `bg-light` | 現状クラスなし | 必要なら `card-main-light`（`bg`=`--bs-light`、字=`--bs-body-color`）を **追加検討**。既存の [components の NeutralCard（`card-custom`）](c:\Users\ryone\Desktop\oshi-app\DESIGN.md) で足りるならクラス追加は省略可 |

**重要**: 現状の不整合の中心は `--app-card-secondary-text: var(--bs-light, #fff)`（[assets/styles.css](c:\Users\ryone\Desktop\oshi-app\assets\styles.css) L17–18）と、`.card-main-secondary .text-muted` が白系 `rgba` 固定（L156–162）の組み合わせ。

---

## Step 2: CSS 実装（主変更は `assets/styles.css` のみ）

1. **Secondary の文字色トークン**を `var(--bs-body-color)`（なければフォールバック）へ変更し、Bootstrap の `card bg-secondary` と同じ読み方にする。
2. **`.card-main-secondary .text-muted`** を、白系 `rgba` から **濃い面用**（例: `var(--bs-secondary-color)` または `rgba(0,0,0,.55)` 系だが、ダークテーマでも破綻しないよう `var(--bs-secondary-color)` 優先）へ変更。**`.card-main-primary` 等の白文字カードの `.text-muted` は現状維持**するため、セレクタを **`.card-main-secondary` だけ分離**する。
3. **Warning** をユーザー例どおり白文字にする場合は `--app-card-warning-text` を白系へ。併せて `.card-main-warning .text-muted` を白系トーンに戻すなど、補助色の階調を揃える。**コントラスト未達テーマは QA で「dark 文字に戻す」逃げ道**を DESIGN に1行書く。
4. （任意）**`card-main-light`** を追加する場合は、`:root` に `--app-card-light-bg` / `--app-card-light-text`、`.card-main-light` ブロック、hover/active を他 `card-main-*` と同パターンで追加。利用箇所は「本当に `bg-light` 相当が欲しい画面」だけに限定（grep で `card-main-secondary` のうち「中立説明」用途を見て判断）。

Python 側の `className` は、**Secondary の意味が「補助の薄い面」で正しい限り変更不要**。用途が「強いセカンダリ色＋白文字」だった画面だけ、別クラスへ差し替える判断が必要なら Step 3 で洗い出す。

## Step 3: 画面横断の見た目確認（主要テーマ）

対象: Minty / Flatly / Darkly など **secondary の明度が大きく違うテーマ**を最低3つ。

- [pages/home.py](c:\Users\ryone\Desktop\oshi-app\pages\home.py)、[features/review/components.py](c:\Users\ryone\Desktop\oshi-app\features\review\components.py)、[pages/settings/index.py](c:\Users\ryone\Desktop\oshi-app\pages\settings\index.py) など `card-main-secondary` が多い画面で、**見出し・本文・muted・リンク**が読めるか確認。
- `card-main-primary` / `info` / `success` / `danger` は白文字のまま崩れていないか（ダークテーマ含む）。
